# Archive - Historical Documentation

This directory contains historical documentation from completed migrations, refactors, and planning sessions.

**Purpose**: Keep root clean while preserving project history.

## Contents

### Migrations
- `MIGRATION_I18N_COPIES.md` - i18n system migration docs
- `MIGRATION_LEGACY_COMPLETE.md` - Legacy code migration completion

### Theme Variants Planning
- `THEME_VARIANTS_PLAN.md` - Initial planning
- `THEME_VARIANTS_IMPLEMENTATION.md` - Implementation details
- `THEME_VARIANTS_MINIMAL.md` - Minimal approach
- `THEME_VARIANTS_SCALABILITY.md` - Scalability analysis

### Testing & Planning
- `TESTING_RESULTS.md` - Historical test results
- `saas-credit.plan.md` - Credit system planning
- `build-before.log` - Build log from pre-migration state

## When to Archive

Move documents here when:
- ✅ Migration/refactor is complete
- ✅ Planning has been implemented
- ✅ Document is no longer actively referenced
- ✅ Information is preserved in Git history

## When NOT to Archive

Keep in root or active docs if:
- ❌ Still being actively worked on
- ❌ Referenced frequently in development
- ❌ Contains current architecture decisions
